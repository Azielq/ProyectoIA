3 Flujo de Datos entre Componentes

Flujo A: Upload y Transcripcion

Recorrido: Usuario → Interfaz → Orquestador → Preprocesado → Motor de Inferencia (Google STT) → Almacen de Datos
# Origen Destino Datos Descripcion
1 Usuario Interfaz Archivo MP3 El usuario selecciona un archivo MP3 desde su dispositivo
2 Interfaz Orquestador POST /upload (multipart/form-data) script.js envia el archivo via FormData al endpoint Flask
3 Orquestador Preprocesado transcribe_audio(path, output) app.py guarda el MP3 en uploads/ y llama a LoadAudio.py
4 Preprocesado Motor IA WAV chunks (55s, LINEAR16, 16kHz) Cada chunk se envia a Google Speech-to-Text via recognize()
5 Motor IA Preprocesado Transcripciones parciales Google retorna el texto reconocido por cada chunk
6 Preprocesado Almacen Transcripcion completa Las transcripciones parciales se concatenan y guardan en call.txt
7 Orquestador Interfaz JSON {message, transcript_length} Flask retorna confirmacion al frontend

Flujo B: Consulta al Chat

Recorrido: Usuario → Interfaz → Orquestador → Motor de Inferencia (GPT-4o) + Postprocesado + Base de Conocimiento
# Origen Destino Datos Descripcion
1 Usuario Interfaz Pregunta en texto El usuario escribe una pregunta y presiona Send
2 Interfaz Orquestador GET /ask?question=... script.js envia la pregunta al endpoint Flask
3 Orquestador Almacen Lee call.txt app.py carga la transcripcion desde el filesystem
4 Orquestador Base Conocim. SYSTEM_PROMPT, API Key Lee las reglas de analisis y credenciales de configuracion
5 Orquestador Motor IA Transcripcion + pregunta + prompt Envia a OpenAI GPT-4o via chat.completions.create()
6 Motor IA Postprocesado Respuesta del modelo GPT-4o retorna el analisis, se formatea como JSON
7 Orquestador Interfaz JSON {answer} Flask retorna la respuesta formateada al frontend
